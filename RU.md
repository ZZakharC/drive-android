Android-клиент для системы облачного хранения **Drive**.
Предоставляет мобильный доступ к управлению файлами, аутентификации и пользовательским аккаунтам через backend API Drive.

[Репозитории с веб-клиентом и сервером](https://github.com/ZZakharC/drive)

---

## 🚀 Возможности

### 🔐 Аутентификация

* Вход и регистрация пользователей
* Сессионная аутентификация (через серверную часть)
* Сохранение токена/сессии
* Поддержка автоматического входа

### 📁 Файловый менеджер

* Просмотр папок и файлов
* Загрузка файлов с устройства
* Скачивание файлов на устройство
* Удаление файлов (в зависимости от прав доступа)
* Визуализация типов файлов (иконки для разных форматов)

### 👥 Система пользователей

* Доступ к профилю пользователя
* Ролевая модель прав доступа (R / C / A)
* Административные действия (при наличии соответствующих прав)

### 📡 Интеграция с сервером

* Прямое взаимодействие с backend-сервером Drive
* Поддержка HTTPS (включая самоподписанные сертификаты)
* JSON-формат обмена данными через API

## ⚙️ Установка и настройка

### 📌 Требования

* Android Studio
* Android SDK 24+
* Запущенный backend-сервер Drive

---

### Настройка URL backend-сервера

Отредактируйте файл:

```
app/src/main/java/org/example/drive/Config.java
```

Пример:

```java
public class Config {
    public static final String SERVER_URL = "https://192.168.1.10:3000";
}
```

---

### Запуск приложения

* Запустите backend-сервер Drive
* Откройте Android-приложение
* Войдите или зарегистрируйтесь
* Начните управление файлами

## 🔐 Модель прав доступа

Используется битовая маска на стороне backend:

| Бит | Разрешение        |
| --- | ----------------- |
| 1   | Чтение (R)        |
| 2   | Изменение (C)     |
| 4   | Администратор (A) |

Клиент применяет ограничения интерфейса на основе полученных прав.

---

## 🔒 Примечания по безопасности

* Сессионная аутентификация (управляется сервером)
* Поддержка HTTPS
* Локально не хранится чувствительная информация (кроме токена сессии)
* Проверка прав доступа выполняется на стороне сервера
